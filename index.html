Proyecto: Compresión de Imágenes y Sistema de Clasificación

Resumen del Proyecto

Este proyecto académico integral abarcó tres áreas principales de procesamiento de imágenes e inteligencia artificial. Implementé desde cero un algoritmo de compresión basado en la Transformada Coseno Discreta (DCT) que logró superar al estándar JPEG en ciertos escenarios. Además, desarrollé y evalué múltiples modelos de Machine Learning para clasificación de dígitos manuscritos, mejorando significativamente los resultados mediante el uso del descriptor HOG. Finalmente, creé un generador de mosaicos fotográficos que aplica estos conceptos de manera creativa.

Parte 1: Compresión de Imágenes con DCT

Desafío

Implementar un algoritmo de compresión de imágenes que supere al estándar JPEG en términos de relación compresión-calidad (PSNR) para ciertas configuraciones.

Implementación

Desarrollé desde cero un códec completo que incluye:

  • Conversión del espacio de color RGB a YCbCr
  • División de la imagen en bloques 8x8 píxeles
  • Aplicación de la Transformada Coseno Discreta (DCT) a cada bloque
  • Cuantización utilizando matrices personalizadas
  • Codificación mediante patrón zig-zag y RLE (Run-Length Encoding)
  • Almacenamiento eficiente usando pickle

Optimización

En lugar de usar la matriz de cuantización estándar de JPEG, implementé un sistema de búsqueda aleatoria para encontrar matrices más eficientes definidas como Q(i,j) = a + b(i+j). Realicé 200 iteraciones con valores aleatorios de a (3-100) y b (1-40).

Resultados Clave

Comparación con JPEG

Encontré 3 configuraciones que superaron a JPEG:

  • a=17, b=3: PSNR=24.65 dB, Ratio=6.83
  • a=24, b=2: PSNR=24.44 dB, Ratio=7.43
  • a=4, b=6: PSNR=24.44 dB, Ratio=6.24

Frente al resultado JPEG: PSNR=24.41 dB, Ratio=5.79

Análisis Visual

Aunque las diferencias visuales son mínimas, las métricas cuantitativas demuestran una mejora significativa. El algoritmo logra mayor compresión manteniendo o mejorando la calidad de imagen.

Este resultado muestra que la matriz de cuantización estándar de JPEG no es universalmente óptima y puede mejorarse para imágenes específicas.

Parte 2: Clasificación de Dígitos Manuscritos

Desafío

Desarrollar y evaluar modelos de Machine Learning para clasificar dígitos manuscritos del dataset MNIST, comparando el uso de píxeles en bruto versus características HOG.

Metodología

Utilicé el conjunto de datos MNIST con 70,000 imágenes de 28x28 píxeles. Implementé y comparé tres algoritmos:

  • K-Nearest Neighbors (KNN) con k=3
  • Árbol de Decisión con criterio Gini
  • SVM con kernel RBF (C=5, gamma=0.05)

Evalué los modelos mediante validación cruzada de 5 folds para obtener resultados robustos.

Extracción de Características HOG

Implementé el descriptor Histogram of Oriented Gradients (HOG) que captura la estructura y forma de los dígitos mediante la distribución de gradientes locales. Esto proporciona características más robustas e invariantes a pequeñas transformaciones.

Resultados

Comparación de Modelos (Precisión Promedio)

Modelo Sin HOG Con HOG Mejora
KNN (k=3) 97.12% 97.37% +0.25%
Árbol de Decisión (Gini) 87.01% 86.87% -0.14%
SVM (RBF) 98.31% 98.94% +0.63%

Análisis de Resultados

El SVM con kernel RBF demostró ser el modelo más efectivo, alcanzando una precisión del 98.94% con características HOG. El uso de HOG mejoró consistentemente los resultados para KNN y SVM, mientras que para Árboles de Decisión no mostró beneficios significativos.

El descriptor HOG proporciona características más compactas y menos ruidosas que los píxeles en bruto, lo que explica la mejora en modelos basados en distancias como KNN y SVM.

Visualización HOG

La aplicación de HOG a una imagen PPM permitió visualizar cómo este descriptor captura los bordes y la estructura de la imagen, confirmando su utilidad para tareas de clasificación donde la forma es más importante que los valores exactos de píxeles.

Parte 3: Generación de Mosaicos Fotográficos

Desafío

Crear un mosaico compuesto por miniaturas (thumbnails) que preserve la estructura y apariencia de una imagen de referencia de gran tamaño.

Implementación

Dividí una imagen grande (Tokyo.jpg) en bloques de 32x32 píxeles y utilicé el dataset CIFAR-10 como fuente de miniaturas. Implementé y comparé tres enfoques para encontrar la miniatura más similar a cada bloque:

  • Media RGB: Calcula el color promedio de cada bloque y miniatura
  • Histograma HSV: Utiliza la distribución de colores en el espacio HSV
  • Combinado: Combina media RGB con histograma HSV y ajuste de color

Para la comparación, utilicé distancias Euclidiana y Cityblock (L1).

Evaluación

Comparé los resultados utilizando tres métricas objetivas:

  • PSNR (Peak Signal-to-Noise Ratio): Mide la similitud pixel a pixel
  • SSIM (Structural Similarity): Evalúa la similitud estructural
  • ΔE (Delta E): Mide la diferencia perceptual de color

Resultados

Método PSNR SSIM ΔE (avg)
Mosaic - Mean RGB 11.531 0.117 19.010
Mosaic - HSV Histogram 10.022 0.145 22.124
Mosaic - Combined (RGB+HSV+Adjust) 11.622 0.131 18.411

Análisis de Resultados

El método combinado (RGB+HSV+Adjust) obtuvo el mejor PSNR y la menor diferencia de color (ΔE), demostrando ser el más fiel a la imagen original. Aunque el método HSV obtuvo el mejor SSIM, su rendimiento visual fue menos natural debido a la priorización de la estructura sobre la fidelidad cromática.

El ajuste de color aplicado en el método combinado fue crucial para mejorar la correspondencia cromática entre los bloques originales y las miniaturas seleccionadas.

Conclusión Visual

La evaluación visual confirmó los resultados cuantitativos: el mosaico generado con el método combinado presenta la mejor combinación de detalle estructural y fidelidad cromática, creando una representación más reconocible de la imagen original.

Conclusiones Generales

Compresión de Imágenes

La implementación personalizada del algoritmo de compresión DCT demostró que es posible superar al estándar JPEG optimizando la matriz de cuantización. Los resultados sugieren que matrices adaptativas podrían ofrecer mejoras significativas sobre el enfoque estándar.

Clasificación con ML

El uso de características HOG mejora consistentemente el rendimiento de modelos de clasificación basados en distancias como KNN y SVM. El SVM con kernel RBF se confirmó como el algoritmo más efectivo para esta tarea, alcanzando precisiones cercanas al 99%.

Mosaicos Fotográficos

La combinación de múltiples descriptores (RGB + HSV) con ajuste de color produce los mejores resultados en la generación de mosaicos, equilibrando fidelidad estructural y cromática. Este enfoque híbrido supera a métodos que utilizan descriptores individuales.

Próximos Pasos

Como extensiones futuras de este trabajo, se podría explorar el uso de matrices de cuantización adaptativas basadas en el contenido de la imagen, la aplicación de redes neuronales convolucionales para la clasificación de dígitos, y la integración de criterios semánticos en la generación de mosaicos.
